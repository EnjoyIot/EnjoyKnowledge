## 🟡 共识但细节分歧（需要你定）

### D1：元数据字段集合

| codex 说 | claude 说 | 我的判断 |
|---|---|---|
| `appliesTo` + `type`（`convention/gotcha/invariant`）+ `tags` | `scope` + `priority` + `appliesTo` | **两者都包含 `appliesTo`**；`type` 分类 vs `priority` 优先级不冲突可并存 |
| 不需要 `priority`（3 级就够） | 需要 `priority`（冲突时优先信谁） | **codex 偏简化，claude 偏完整** |

**建议合并**：
```yaml
---
id: rust-no-unwrap          # codex 强调
appliesTo: [rust, glob:**/*.rs]  # 两个共识
priority: 3                 # claude 建议（1=低, 5=高）
type: gotcha                # codex 建议（convention/gotcha/invariant）
tags: [rust, error-handling] # 两个共识
scope: project              # 两个共识（project/user/team）
---
```

### D2：命名空间 vs 目录层级

| codex | claude | 我的判断 |
|---|---|---|
| **平铺** `rules/<id>.md`，分类用 `tags` | **分层** `rules/rust/<topic>.md` | 取决于团队规模 |

**小团队（< 10 rules）→ 平铺 + tags**（codex）

**MVP 选平铺**——30+ 时再分层。

## 🔴 真正分歧（我无法独立决定）

### 分歧 1：YAML frontmatter 的角色

| codex | claude |
|---|---|
| **YAML 是必需的**（"OKF frontmatter"作为索引） | **YAML 是可选的**（"仅当 rule 需要被多工具差异化渲染时才加"） |

**含义**：
- codex：所有 rule 都带 frontmatter（id, appliesTo, priority, type, tags）
- claude：大部分 rule 可以纯 markdown，frontmatter 只在需要差异化时加

**我的判断**：**claude 的方案更适合 coding 场景**——大部分 rule 是不分工具的通用规范（"用 2 空格缩进"），只有少数 rule 需要"只在 Trae 生效"。强制 frontmatter 增加噪音。

### 分歧 2：managed section 标记的格式

| codex | claude |
|---|---|
| `<!-- ENJOYKNOWLEDGE:START -->` | `<!-- enjoyknowledge:managed:start -->` |
| 单段（start/end 包整块） | 单段（managed:start/end） |

**两者等价**——选哪个看团队偏好。建议用 **`<!-- ek:managed:start -->` / `<!-- ek:managed:end -->`**（`ek` 短前缀易搜索）。

### 分歧 3：体积上限

| codex | claude | 共识点 |
|---|---|---|
| 5000 词总 / 150 词单条 | ~300 行总 / > 500 行丢 | **都需要上限** |

**我的判断**：**取保守值——4000 词总 / 100 词单条**，避免 5000 边界争议。

## 📋 整合后的设计（最终方案）

### 目录结构

```
<project_root>/
├── .enjoyknowledge/
│   └── rules/                       # SoT（唯一规则源）
│       ├── coding-style.md          # 纯 markdown，frontmatter 可选
│       ├── naming.md
│       ├── rust-unwrap.md           # 带 frontmatter（指定 appliesTo=rust）
│       └── archive/                 # 不被 sync 包含（doctor 引导归档）
├── CLAUDE.md                        # Claude Code 配置（managed section）
├── AGENTS.md                        # Codex 配置（managed section）
├── .cursorrules                     # Cursor 配置（managed section）
├── .traerules                       # Trae 配置（managed section）
└── ...                              # 其他 5 个工具
```

### SoT 格式

```markdown
<!-- 普通规则：纯 markdown，frontmatter 可选 -->
# Rust 错误处理规范
所有 `Result` 必须显式处理，`?` 传播必须配合 `anyhow::Context`。

<!-- 需要差异化时：带 frontmatter -->
---
id: rust-no-unwrap
appliesTo: [rust, glob:**/*.rs]
priority: 4
type: gotcha
scope: project
---

# 不要在生产代码用 `.unwrap()`
`unwrap()` 只在 prototype 和测试中允许。生产代码用 `?` 或 `expect("reason")`。
```

### 命令清单

```bash
# 主命令
enjoyknowledge rule list                    # 列出所有 rules
enjoyknowledge rule show <id>               # 显示某条 rule
enjoyknowledge rule add <id>                # 新增（启动 editor）
enjoyknowledge rule sync --tool claude      # 渲染到 CLAUDE.md
enjoyknowledge rule sync --tool all         # 渲染到所有已配置工具
enjoyknowledge rule sync --dry-run          # 预览不写

# 校验
enjoyknowledge doctor                       # 检查 managed section 完整性
enjoyknowledge doctor --check rule-size     # 检查体积超限
enjoyknowledge doctor --check mark完整性    # 检查 managed 标记未删
```

### Managed Section 格式

```markdown
# CLAUDE.md（用户文件）

<!-- 用户手写部分：保留 -->
# 我的项目说明
这里是我手写的项目背景。

<!-- ek:managed:start -->
<!-- 以下由 enjoyknowledge rule sync 自动生成，请勿手改 -->
# Coding Style
- 2 空格缩进
- 函数不超过 50 行
# Naming
- snake_case for functions
- PascalCase for types
<!-- ek:managed:end -->

<!-- 用户手写部分：保留 -->
# 我自己的工作流
...
```

### 关键约束（doctor 检查）

| 检查项 | 阈值 | 处理 |
|---|---|---|
| 单条 rule 长度 | > 100 词 | warning |
| 总 ruleset 长度 | > 4000 词 | warning + 建议归档 |
| managed 标记存在 | 缺失 | error |
| managed 标记嵌套 | 嵌套 | error |
| 渲染幂等性 | 二次渲染 diff 非空 | error（renderer bug） |

## 🎯 与现有 `init --ai` 模式的对比

| 维度 | 当前 `init --ai claude` | 新 `rule sync --tool claude` |
|---|---|---|
| 触发时机 | 一次性 | 每次改 rule 后 |
| 写入方式 | 全文件覆盖 | 段内覆盖（managed） |
| 用户手改 | 覆盖丢失 | 段外保留，段内覆盖 |
| 多工具 | 每次只能生成一个 | `--tool all` 一次同步 |
| 幂等 | 不保证 | 强制保证 |
| 回滚 | 无 | `git checkout CLAUDE.md` 即可 |
| 体积约束 | 无 | 4000 词硬上限 |
